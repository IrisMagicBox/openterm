## 一、配置层（5个表）

### 1. hosts - 主机配置表

**存储内容**: SSH 连接配置

| 字段        | 类型    | 说明         | 示例                    |
| ----------- | ------- | ------------ | ----------------------- |
| `id`        | TEXT    | 唯一标识     | `"host-abc123"`         |
| `alias`     | TEXT    | 显示名称     | `"生产服务器"`          |
| `ip`        | TEXT    | IP 地址      | `"192.168.1.100"`       |
| `port`      | INTEGER | SSH 端口     | `22`                    |
| `username`  | TEXT    | 用户名       | `"root"`                |
| `password`  | TEXT    | 密码（可选） | `"encrypted..."`        |
| `keyPath`   | TEXT    | SSH key 路径 | `"~/.ssh/id_rsa"`       |
| `tags`      | TEXT    | 标签（JSON） | `["production", "gpu"]` |
| `createdAt` | INTEGER | 创建时间     | `1713200000000`         |

**作用**:

- 存储所有可连接的远程服务器信息
- 支持密码和 SSH key 两种认证方式
- 本地主机也有记录（id = "local"）

**使用场景**:

```typescript
// 创建新主机
hostDB.createHost({
  alias: 'AI训练服务器',
  ip: '10.0.0.5',
  username: 'ubuntu',
  keyPath: '~/.ssh/ai-server'
})

// 获取主机列表
const hosts = hostDB.getAllHosts()
```

### 6. command_patterns - 命令模式表 ⭐

**存储内容**: 命令信任度学习

| 字段             | 类型    | 说明             | 示例            |
| ---------------- | ------- | ---------------- | --------------- |
| `id`             | TEXT    | 唯一标识         | `"pattern-123"` |
| `hostId`         | TEXT    | 关联主机         | `"host-abc"`    |
| `commandPattern` | TEXT    | 命令模式（正则） | `"^ls -la$"`    |
| `approvalCount`  | INTEGER | 批准次数         | `10`            |
| `rejectionCount` | INTEGER | 拒绝次数         | `0`             |
| `trustLevel`     | TEXT    | 信任等级         | `"trusted"`     |
| `lastSeen`       | INTEGER | 最后出现时间     | `1713200000000` |
| `createdAt`      | INTEGER | 创建时间         | `1713100000000` |

**作用**:

- **学习用户习惯**: 记录哪些命令被批准/拒绝
- **自动授权**: 信任度高的命令自动执行，无需确认
- **安全策略**: `untrusted` → `familiar` → `trusted`

**使用场景**:

```typescript
// 用户多次批准 "ls -la" 后
commandPatternDB.incrementApproval(hostId, '^ls -la$')

// 检查命令信任度
const trustLevel = commandPatternDB.getTrustLevel(hostId, command)
if (trustLevel === 'trusted') {
  // 自动执行，无需确认
} else {
  // 请求用户确认
}
```

## 二、会话层（3个表）

### 7. topics - 话题表 ⭐⭐

**存储内容**: 对话会话（类似 ChatGPT 的会话）

| 字段                 | 类型    | 说明                 | 示例                   |
| -------------------- | ------- | -------------------- | ---------------------- |
| `id`                 | TEXT    | 唯一标识             | `"topic-xyz789"`       |
| `title`              | TEXT    | 话题标题             | `"部署应用到生产环境"` |
| `hostIds`            | TEXT    | 关联主机（JSON数组） | `["host-1", "host-2"]` |
| `selectedProviderId` | TEXT    | 当前使用的 AI 提供商 | `"openai"`             |
| `selectedModelId`    | TEXT    | 当前使用的模型       | `"gpt-4"`              |
| `lastMessageAt`      | INTEGER | 最后消息时间         | `1713200000000`        |
| `createdAt`          | INTEGER | 创建时间             | `1713100000000`        |

**作用**:

- **会话隔离**: 每个话题独立，有自己的消息历史
- **多主机关联**: 一个话题可以关联多个主机（如同时操作生产和测试环境）
- **模型绑定**: 每个话题可以独立选择 AI 模型

**使用场景**:

```typescript
// 创建新话题
topicDB.createTopic({
  title: '排查生产环境问题',
  hostIds: ['host-production', 'host-monitoring']
})

// 获取话题的所有终端
const sessions = agentService.getSessions(topicId)
```

### 8. messages - 消息表 ⭐⭐⭐

**存储内容**: 对话历史（最重要）

| 字段         | 类型    | 说明              | 示例                                          |
| ------------ | ------- | ----------------- | --------------------------------------------- |
| `id`         | TEXT    | 唯一标识          | `"msg-123"`                                   |
| `topicId`    | TEXT    | 关联话题          | `"topic-xyz"`                                 |
| `role`       | TEXT    | 角色              | `"user"`, `"assistant"`, `"system"`, `"tool"` |
| `content`    | TEXT    | 内容              | `"帮我部署应用"`                              |
| `timestamp`  | INTEGER | 时间戳            | `1713200000000`                               |
| `thought`    | TEXT    | 思考过程（Agent） | `"我需要先检查环境..."`                       |
| `toolCalls`  | TEXT    | 工具调用（JSON）  | `[{name: "execute_command", ...}]`            |
| `toolCallId` | TEXT    | 关联工具调用ID    | `"call-abc"`                                  |
| `name`       | TEXT    | 工具名称          | `"execute_command"`                           |

**作用**:

- **对话历史**: 存储用户和 Agent 的所有对话
- **Agent 思考**: 记录 Agent 的推理过程（CoT）
- **工具调用**: 记录 Agent 调用了哪些工具
- **上下文来源**: AgentRunner 从此表读取历史构建上下文

**消息类型详解**:

| role        | 说明       | 示例                    |
| ----------- | ---------- | ----------------------- |
| `user`      | 用户输入   | `"帮我查看日志"`        |
| `assistant` | Agent 回复 | `"我来帮您查看日志..."` |
| `system`    | 系统提示   | 系统指令、终端上下文    |
| `tool`      | 工具结果   | 命令执行结果            |

**使用场景**:

```typescript
// 获取话题的所有消息（用于构建 Agent 上下文）
const messages = messageDB.getMessages(topicId)

// Agent 回复后存储
messageDB.createMessage({
  topicId: 'topic-xyz',
  role: 'assistant',
  content: '已为您查看日志，发现错误...',
  thought: '我执行了 tail -f 命令...'
})
```

### 9. memories - 记忆表 ⭐⭐

**存储内容**: 长期记忆（Agent 学习的内容）

| 字段         | 类型    | 说明             | 示例                                     |
| ------------ | ------- | ---------------- | ---------------------------------------- |
| `id`         | TEXT    | 唯一标识         | `"memory-123"`                           |
| `type`       | TEXT    | 记忆类型         | `"habit"`, `"host_fact"`, `"experience"` |

| `hostId`     | TEXT    | 关联主机（可选） | `"host-abc"`                             |
| `topicId`    | TEXT    | 关联话题（可选） | `"topic-xyz"`                            |
| `importance` | INTEGER | 重要性 1-5       | `4`                                      |
| `timestamp`  | INTEGER | 时间戳           | `1713200000000`                          |

**记忆类型详解**:

| type         | 说明     | 示例                                   |
| ------------ | -------- | -------------------------------------- |
| `habit`      | 用户习惯 | `"用户喜欢用 htop 查看负载"`           |

**使用场景**:

```typescript
// 任务完成后自动提取记忆
MemoryManager.reflectOnTask(taskId)

// Agent 思考前召回记忆
const context = await MemoryManager.recallRelevantContext(topicId, '部署应用')
// 返回: "[主机事实]: 主机使用 Ubuntu 22.04\n[用户习惯]: 喜欢用 docker-compose"
```

## 三、终端层（2个表）⭐⭐⭐

### 10. terminal_sessions - 终端会话表

**存储内容**: 终端元数据（不是内容！）

| 字段                    | 类型    | 说明           | 示例                                  |
| ----------------------- | ------- | -------------- | ------------------------------------- |
| `id`                    | TEXT    | 唯一标识       | `"session-abc123"`                    |
| `topicId`               | TEXT    | 关联话题       | `"topic-xyz"`                         |
| `hostId`                | TEXT    | 关联主机       | `"host-abc"`                          |
| `hostAlias`             | TEXT    | 主机别名       | `"生产服务器"`                        |
| `name`                  | TEXT    | 终端名称       | `"终端-1"`                            |
| `status`                | TEXT    | 状态           | `"active"`, `"streaming"`, `"closed"` |
| `shellType`             | TEXT    | Shell 类型     | `"bash"`, `"zsh"`                     |
| `shellIntegrationReady` | INTEGER | Shell 集成就绪 | `1`                                   |
| `createdAt`             | INTEGER | 创建时间       | `1713200000000`                       |
| `closedAt`              | INTEGER | 关闭时间       | `null`                                |

**作用**:

- **终端注册**: 记录有哪些终端存在
- **生命周期**: 追踪终端创建、活跃、关闭状态
- **归属关系**: 终端属于哪个话题和主机

**注意**: 这个表**不存储**终端的内容（命令和输出），只存储元数据！

**使用场景**:

```typescript
// 创建新终端
const session = await agentService.createTerminal(topicId, hostId, '调试终端')

// 获取话题的所有终端
const sessions = await agentService.getSessions(topicId)
// 返回: [{id: 'session-1', name: '调试终端', status: 'active', ...}]
```

### 11. terminal_io - 终端 I/O 表 ⭐⭐⭐⭐⭐

**存储内容**: 终端的所有输入和输出（最核心！）

| 字段             | 类型    | 说明         | 示例                            |
| ---------------- | ------- | ------------ | ------------------------------- |
| `id`             | TEXT    | 唯一标识     | `"io-123"`                      |
| `sessionId`      | TEXT    | 关联终端会话 | `"session-abc"`                 |
| `topicId`        | TEXT    | 关联话题     | `"topic-xyz"`                   |
| `hostId`         | TEXT    | 关联主机     | `"host-abc"`                    |
| `type`           | TEXT    | I/O 类型     | `"input"` 或 `"output"`         |
| `source`         | TEXT    | 来源         | `"agent"`, `"user"`, `"system"` |
| `content`        | TEXT    | 内容         | `"ls -la"` 或输出文本           |
| `exitCode`       | INTEGER | 退出码       | `0`, `1`, `130`                 |
| `durationMs`     | INTEGER | 执行时长     | `5000`                          |
| `relatedInputId` | TEXT    | 关联输入ID   | `"io-122"`                      |
| `isStreaming`    | INTEGER | 是否流式     | `0`                             |
| `chunkIndex`     | INTEGER | 块索引       | `0`                             |
| `isTruncated`    | INTEGER | 是否截断     | `0`                             |
| `cwd`            | TEXT    | 工作目录     | `/home/user/project`            |
| `taskId`         | TEXT    | 关联任务     | `"task-456"`                    |
| `stepId`         | TEXT    | 关联步骤     | `"step-789"`                    |
| `timestamp`      | INTEGER | 时间戳       | `1713200000000`                 |

**字段详解**:

#### type - I/O 类型

| 值       | 说明       | 示例                         |
| -------- | ---------- | ---------------------------- |
| `input`  | 输入的命令 | `"ls -la"`, `"cd /home"`     |
| `output` | 命令的输出 | `"total 128\ndrwxr-xr-x..."` |

#### source - 来源

| 值       | 说明       | 场景               |
| -------- | ---------- | ------------------ |
| `agent`  | Agent 执行 | Agent 自动执行命令 |
| `user`   | 用户输入   | 用户手动输入       |
| `system` | 系统消息   | 系统通知、错误     |

#### exitCode - 退出码

| 值    | 含义                 |
| ----- | -------------------- |
| `0`   | 成功                 |
| `1`   | 一般错误             |
| `130` | Ctrl+C 中断 (SIGINT) |
| `137` | SIGKILL 终止         |
| `255` | SSH 错误             |

#### isStreaming - 流式输出

| 值  | 说明     | 示例                             |
| --- | -------- | -------------------------------- |
| `0` | 普通命令 | `ls`, `cat`                      |
| `1` | 持续输出 | `tail -f`, `watch`, `nvidia-smi` |

**作用**:

- **完整记录**: 记录终端的所有操作历史
- **审计追踪**: 谁（source）在什么时间（timestamp）执行了什么（content）
- **结果查询**: 查询命令是否成功（exitCode）、执行多久（durationMs）
- **上下文构建**: Agent 通过查询此表了解终端历史

**使用场景**:

```typescript
// 查询终端的所有 I/O
const io = terminalIODB.getTerminalIO(sessionId)

// 查询最近 10 条命令
const recentCommands = terminalIODB.getRecentCommands(sessionId, 10)

// 查询特定任务的 I/O
const taskIO = terminalIODB.getTaskIO(taskId)

// Agent 构建终端上下文时查询
const context = commandExecutor.buildTerminalContext(topicId)
// 内部实现: SELECT * FROM terminal_io WHERE topicId = ? ORDER BY timestamp DESC
```

## 四、任务层（4个表）

### 12. tasks - 任务表

**存储内容**: 用户请求的任务

| 字段                 | 类型    | 说明             | 示例                                   |
| -------------------- | ------- | ---------------- | -------------------------------------- |
| `id`                 | TEXT    | 唯一标识         | `"task-abc123"`                        |
| `topicId`            | TEXT    | 关联话题         | `"topic-xyz"`                          |
| `title`              | TEXT    | 标题             | `"部署应用"`                           |
| `goal`               | TEXT    | 目标             | `"将应用部署到生产环境"`               |
| `status`             | TEXT    | 状态             | `"running"`, `"completed"`, `"failed"` |
| `summary`            | TEXT    | 总结             | `"成功部署，访问地址: ..."`            |
| `selectedProviderId` | TEXT    | 使用的 AI 提供商 | `"openai"`                             |
| `selectedModelId`    | TEXT    | 使用的模型       | `"gpt-4"`                              |
| `createdAt`          | INTEGER | 创建时间         | `1713200000000`                        |
| `updatedAt`          | INTEGER | 更新时间         | `1713205000000`                        |

**作用**:

- **任务追踪**: 记录每个用户请求的执行状态
- **结果汇总**: 任务完成后的总结
- **成本追踪**: 记录使用的 AI 模型（用于统计）

**状态流转**:

```
pending → running → completed
              ↓
            failed
```

## 六、核心数据流示例

### 场景: 用户说"帮我查看日志"

```
1. 用户输入
   └──► messages 表新增记录
        role: 'user'
        content: '帮我查看日志'

2. AgentRunner 处理
   └──► tasks 表新增记录
        status: 'running'

3. Agent 决定执行命令
   └──► task_steps 表新增记录
        type: 'command'
        content: 'tail -f /var/log/app.log'

4. 创建/获取终端
   └──► terminal_sessions 表（如需要新建）

5. 执行命令
   └──► terminal_io 表新增 2 条记录
        记录 1: type='input', content='tail -f /var/log/app.log'
        记录 2: type='output', content='...日志内容...', exitCode=0

6. 更新步骤
   └──► task_steps 表更新
        status: 'completed'
        rawOutput: '...日志内容...'

7. Agent 回复
   └──► messages 表新增记录
        role: 'assistant'
        content: '日志显示...'

8. 任务完成
   └──► tasks 表更新
        status: 'completed'
        summary: '成功查看日志，发现...'

9. 提取记忆（异步）
   └──► memories 表新增记录
        type: 'host_fact'
        content: '应用日志位于 /var/log/app.log'
```

## 七、关键问题分析

### 问题 1: terminal_io 表数据量爆炸

**现状**:

- 每个命令产生 2 条记录（input + output）
- 输出可能很大（被截断前可达 100KB）
- 长时间运行的流式命令产生大量 chunk

**影响**:

- 数据库体积快速增长
- 查询性能下降
- 上下文组装时读取历史变慢

**已有缓解措施**:

- `isTruncated`: 大输出被截断，只存前 2000 字符
- `MemoryManager.distillObservation()`: 大输出自动提纯
- 但缺乏归档/清理机制

### 问题 2: 终端状态不透明

**现状**:

- 只能通过 `terminal_io` 反推终端状态
- 需要查询最近 N 条记录才能知道"当前在干嘛"
- 长时间运行任务的状态无法实时获取

**影响**:

- Agent 容易"忘记"终端状态
- 无法快速回答"这个终端在运行什么"
- 监控类任务状态不透明

### 问题 3: 缺乏终端语义标签

**现状**:

- 终端只有 `name` 字段（如"终端-1"）
- 没有 `purpose`, `tags` 等语义化字段
- 无法实现"在 monitor 终端查看 GPU"

**影响**:

- 多终端场景管理困难
- 模糊指令无法智能匹配
- 用户体验不够智能

### 问题 4: 记忆召回粒度粗

**现状**:

- 记忆只有三种类型: habit, host_fact, experience
- 没有终端级别的记忆
- 无法记录"终端1正在运行训练任务，进度 45%"

**影响**:

- Agent 对终端的"短期记忆"不足
- 长对话后容易忘记终端状态
- 无法做智能提醒（如"这个终端有任务在运行，确定要执行新命令吗？"）

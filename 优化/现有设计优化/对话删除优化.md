# OpenTerm 对话删除优化方案

## 一、问题定义

### 1.1 当前删除行为的问题

根据现有架构分析，OpenTerm 当前的数据删除策略存在以下问题：

| 删除操作          | 影响的数据                                                          | 问题                                          |
| ----------------- | ------------------------------------------------------------------- | --------------------------------------------- |
| **删除 Topic**    | topics, messages, terminal_sessions, terminal_io, tasks, task_steps | CASCADE 级联删除，除 memories 外全部丢失      |
| **删除 Terminal** | terminal_sessions, terminal_io                                      | terminal_io 被 CASCADE 删除，命令历史永久丢失 |
| **删除 Host**     | terminal_sessions(hostId→NULL), terminal_io(hostId→NULL)            | 数据保留但失去关联，查询困难                  |

### 1.2 核心矛盾

**用户需求与当前设计的冲突：**

- **Topic 作为大隔离边界**：删除 Topic 时应该可以清理该 Topic 下的所有临时数据
- **历史记录需要保留**：删除 Topic 中的终端或主机时，操作记录、命令、对话（池子中的数据）不应被物理删除
- **模型需要访问历史**：AI 模型应该能够搜索和查看已删除实体的历史记录

### 1.3 用户明确需求

> "当删除 topic 时，全部删除但保留记忆这个没问题"
>
> "当删除 topic 中的主机或者终端时，他们的操作记录、命令、对话、也就是池子中的东西不能删除"
>
> "Topic 为大隔离，当你建立 topic 后，你操作过的主机、终端，都应该有记录"
>
> "模型正常来看就像是删除了一样，给予模型适当的搜索工具以查看历史做过什么"

## 二、设计目标

### 2.1 核心设计原则

| 原则                | 说明                                                                                                  |
| ------------------- | ----------------------------------------------------------------------------------------------------- |
| **Topic 级联删除**  | 删除 Topic 时，可以 CASCADE 删除其下的 messages, terminal_sessions, tasks 等临时数据，但保留 memories |
| **Terminal 软删除** | 删除 Topic 中的终端时，使用逻辑删除，保留 terminal_io 中的所有命令历史                                |
| **Host 软删除**     | 删除 Topic 中的主机时，使用逻辑删除，保留所有关联的操作记录                                           |
| **历史可追溯**      | 所有操作、命令、对话（pool 数据）必须完整保留，支持审计和回溯                                         |
| **模型感知**        | AI 模型正常视角下已删除的项目不可见，但提供专门工具搜索历史                                           |
| **Topic 边界**      | Topic 是主要隔离边界，同一 Topic 内的所有操作都有完整记录                                             |

### 2.2 数据保留矩阵

| 数据类型              | Topic 删除 | Terminal 删除 | Host 删除 |
| --------------------- | ---------- | ------------- | --------- |
| **messages**          | ❌ 删除    | ✅ 保留       | ✅ 保留   |
| **terminal_sessions** | ❌ 删除    | ⚠️ 软删除     | ✅ 保留   |
| **terminal_io**       | ❌ 删除    | ✅ 保留       | ✅ 保留   |
| **tasks**             | ❌ 删除    | ✅ 保留       | ✅ 保留   |
| **task_steps**        | ❌ 删除    | ✅ 保留       | ✅ 保留   |
| **memories**          | ✅ 保留    | ✅ 保留       | ✅ 保留   |

### 3.2 软删除标记策略

采用**三字段标记法**：

| 字段        | 类型    | 说明                              |
| ----------- | ------- | --------------------------------- | ------- | ---------- |
| `isDeleted` | INTEGER | 是否已删除 (0 = 正常, 1 = 已删除) |
| `deletedAt` | INTEGER | 删除时间戳（毫秒）                |
| `deletedBy` | TEXT    | 删除者标识（'user'                | 'agent' | 'system'） |

**优点**：

- 可恢复：保留完整删除信息，支持撤销
- 可审计：记录删除时间和操作者
- 可筛选：默认查询 WHERE isDeleted = 0

```sql
  source TEXT NOT NULL,
  content TEXT NOT NULL,

FROM terminal_io;

-- 删除旧表，重命名新表
DROP TABLE terminal_io;
ALTER TABLE terminal_io_v2 RENAME TO terminal_io;

CREATE INDEX idx_terminal_io_session ON terminal_io(sessionId, isDeleted);
```

### 5.2 删除逻辑实现

#### 5.2.1 Terminal 软删除流程

```typescript
class TerminalService {
  async softDeleteTerminal(
    sessionId: string,
    deletedBy: 'user' | 'agent' | 'system'
  ): Promise<void> {
    const now = Date.now()

    // 1. 开启事务
    const transaction = db.transaction(() => {
      // 2. 软删除 terminal_session
      terminalSessionDB.softDeleteSession(sessionId, deletedBy)

      // 3. 标记关联的 terminal_io
      terminalIODB.markIOAsDeletedBySession(sessionId, now, deletedBy)

      // 4. 标记关联的 tasks
      taskDB.markTasksAsDeletedByTerminal(sessionId, now, deletedBy)

      // 5. 记录删除事件（用于审计）
      auditLogDB.createLog({
        type: 'terminal_deleted',
        targetId: sessionId,
        deletedBy,
        deletedAt: now,
        topicId: session.topicId
      })
    })

    transaction()

    // 6. 发布事件
    eventBus.publish('terminal:soft-deleted', {
      sessionId,
      topicId: session.topicId,
      deletedBy,
      deletedAt: now
    })
  }
}
```

#### 5.2.2 Terminal 恢复流程

```typescript
class TerminalService {
  async restoreTerminal(sessionId: string): Promise<void> {
    // 1. 检查终端是否存在且已删除
    const session = terminalSessionDB.getSession(sessionId, true)
    if (!session || !session.isDeleted) {
      throw new Error('Terminal not found or not deleted')
    }

    // 2. 检查关联的 host 是否还存在
    if (session.hostId) {
      const host = hostDB.getHost(session.hostId)
      if (!host) {
        throw new Error('Associated host no longer exists')
      }
    }

    // 3. 开启事务恢复
    const transaction = db.transaction(() => {
      // 恢复 terminal_session
      terminalSessionDB.restoreSession(sessionId)

      // 恢复关联的 terminal_io
      terminalIODB.restoreIOBySession(sessionId)

      // 恢复关联的 tasks
      taskDB.restoreTasksByTerminal(sessionId)
    })

    transaction()
  }
}
```

#### 6.2.1 search_terminal_history

```typescript
const searchTerminalHistoryTool = define('search_terminal_history', {
  description: '搜索 Topic 中所有终端（包括已删除）的命令历史',
  parameters: z.object({
    topicId: z.string().describe('Topic ID'),
    query: z.string().describe('搜索关键词，支持命令内容、输出内容模糊匹配'),
    filters: z
      .object({
        hostId: z.string().optional().describe('限制搜索特定主机'),
        terminalName: z.string().optional().describe('限制搜索特定终端名称'),
        startTime: z.number().optional().describe('起始时间戳（毫秒）'),
        endTime: z.number().optional().describe('结束时间戳（毫秒）'),
        includeActive: z.boolean().optional().default(true).describe('是否包含活跃终端'),
        includeDeleted: z.boolean().optional().default(true).describe('是否包含已删除终端')
      })
      .optional(),
    limit: z.number().optional().default(20).describe('返回结果数量限制')
  }),

  async execute(args, ctx) {
    const results = terminalIODB.searchCommandHistory(args.topicId, args.query, {
      hostId: args.filters?.hostId,
      startTime: args.filters?.startTime,
      endTime: args.filters?.endTime,
      includeDeletedSessions: args.filters?.includeDeleted !== false,
      limit: args.limit
    })

    // 格式化输出
    const formatted = results.map((r) => ({
      command: r.io.content,
      output: r.io.type === 'output' ? r.io.content : undefined,
      timestamp: r.io.timestamp,
      terminal: {
        id: r.session?.id || 'deleted',
        name: r.session?.name || '已删除的终端',
        isDeleted: r.isSessionDeleted
      },
      host: {
        id: r.io.hostId,
        alias: r.session?.hostAlias || '未知主机'
      },
      exitCode: r.io.exitCode,
      duration: r.io.durationMs
    }))

    return {
      output: JSON.stringify(
        {
          total: formatted.length,
          results: formatted
        },
        null,
        2
      )
    }
  }
})
```

```typescript
const getTopicTimelineTool = define('get_topic_timeline', {
  description: '获取 Topic 的完整操作时间线，包括活跃和已删除的终端',
  parameters: z.object({
    topicId: z.string(),
    options: z
      .object({
        startTime: z.number().optional(),
        endTime: z.number().optional(),
        includeDeleted: z.boolean().optional().default(true),
        groupBy: z.enum(['terminal', 'host', 'time']).optional().default('time')
      })
      .optional()
  }),

  async execute(args, ctx) {
    const timeline = terminalIODB.getTopicCommandTimeline(args.topicId, {
      startTime: args.options?.startTime,
      endTime: args.options?.endTime,
      includeDeleted: args.options?.includeDeleted
    })

    // 按终端分组
    const groupedByTerminal = _.groupBy(timeline, 'sessionId')

    const summary = Object.entries(groupedByTerminal).map(([sessionId, items]) => {
      const first = items[0]
      return {
        terminalId: sessionId,
        terminalName: first.sessionName,
        hostId: first.hostId,
        hostAlias: first.hostAlias,
        isDeleted: first.isSessionDeleted,
        commandCount: items.filter((i) => i.type === 'input').length,
```

### 7.1 Agent 上下文增强

在 ContextAssembler 中注入已删除终端的概要信息：

```typescript
class ContextAssembler {
  async assemble(): Promise<AssembledContext> {
    const messages: ChatCompletionMessageParam[] = []

    // 1. System Prompt
    messages.push({ role: 'system', content: this.systemPrompt })

    // 2. 终端上下文（活跃终端）
    const terminalContext = this.buildTerminalContext()
    messages.push({ role: 'system', content: terminalContext })

    // 3. 【新增】已删除终端历史概要
    const deletedSummary = await this.buildDeletedTerminalSummary()
    if (deletedSummary) {
      messages.push({ role: 'system', content: deletedSummary })
    }

    // 4. 历史消息
    // ...

    return { messages, budget: this.calculateBudget(messages) }
  }

  async buildDeletedTerminalSummary(): Promise<string | null> {
    // 只获取最近删除的终端（例如最近 7 天）
    const recentDeleted = terminalSessionDB.getRecentlyDeleted(
      this.topicId,
      7 * 24 * 60 * 60 * 1000 // 7 天
    )

    if (recentDeleted.length === 0) return null

    const lines = ['### 已删除终端历史:']
    lines.push('(以下终端已被删除，但命令历史保留，可使用搜索工具查看详情)')
    lines.push('')

    for (const term of recentDeleted.slice(0, 5)) {
      lines.push(`- ${term.name} (${term.hostAlias}) - 删除于 ${formatDate(term.deletedAt)}`)

      // 获取该终端的最后几条命令
      const recentCommands = terminalIODB.getRecentCommands(term.id, 3)
      for (const cmd of recentCommands) {
        lines.push(`  └ 最后命令: ${cmd.content.slice(0, 50)}`)
      }
    }

    lines.push('')
    lines.push('如需查看完整历史，请使用 search_terminal_history 或 view_terminal_io 工具。')

    return lines.join('\n')
  }
}
```

```sql
-- =====================================================
-- OpenTerm 软删除迁移脚本
-- =====================================================

-- 开始事务
BEGIN TRANSACTION;

-- 1. 备份现有表（以防万一）
CREATE TABLE terminal_io_backup AS SELECT * FROM terminal_io;
CREATE TABLE terminal_sessions_backup AS SELECT * FROM terminal_sessions;

-- 2. 添加软删除字段到 terminal_sessions
ALTER TABLE terminal_sessions
ADD COLUMN isDeleted INTEGER DEFAULT 0;

ALTER TABLE terminal_sessions
ADD COLUMN deletedAt INTEGER;

ALTER TABLE terminal_sessions
ADD COLUMN deletedBy TEXT;

-- 3. 创建新的 terminal_io 表（修改外键约束）
CREATE TABLE terminal_io_new (
  id TEXT PRIMARY KEY,
  sessionId TEXT,
  topicId TEXT NOT NULL,
  hostId TEXT,
  type TEXT NOT NULL,
  source TEXT NOT NULL,
  content TEXT NOT NULL,
  exitCode INTEGER,
  durationMs INTEGER,
  relatedInputId TEXT,
  isStreaming INTEGER,
  chunkIndex INTEGER,
  isTruncated INTEGER,
  cwd TEXT,
  taskId TEXT,
  stepId TEXT,
  timestamp INTEGER NOT NULL,
  isDeleted INTEGER DEFAULT 0,
  deletedAt INTEGER,
  deletedBy TEXT,
  FOREIGN KEY (sessionId) REFERENCES terminal_sessions(id) ON DELETE SET NULL,
  FOREIGN KEY (topicId) REFERENCES topics(id) ON DELETE CASCADE
);

-- 4. 迁移数据
INSERT INTO terminal_io_new SELECT
  id, sessionId, topicId, hostId, type, source, content,
  exitCode, durationMs, relatedInputId, isStreaming, chunkIndex,
  isTruncated, cwd, taskId, stepId, timestamp,
  0, NULL, NULL
FROM terminal_io;

-- 5. 删除旧表，重命名新表
DROP TABLE terminal_io;
ALTER TABLE terminal_io_new RENAME TO terminal_io;

-- 6. 创建索引
CREATE INDEX idx_terminal_sessions_topic_active
ON terminal_sessions(topicId, isDeleted)
WHERE isDeleted = 0;

CREATE INDEX idx_terminal_io_session
ON terminal_io(sessionId, isDeleted);

CREATE INDEX idx_terminal_io_topic
ON terminal_io(topicId, isDeleted, timestamp);

CREATE INDEX idx_terminal_io_deleted
ON terminal_io(isDeleted, deletedAt);

-- 7. 验证数据完整性

UNION ALL
SELECT 'terminal_io count', COUNT(*) FROM terminal_io
UNION ALL
SELECT 'deleted sessions', COUNT(*) FROM terminal_sessions WHERE isDeleted = 1
UNION ALL
SELECT 'deleted io', COUNT(*) FROM terminal_io WHERE isDeleted = 1;

-- 提交事务
COMMIT;
```

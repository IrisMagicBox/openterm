# Feedback

OpenTerm is still shaped by real terminal workflows. Useful feedback is concrete, reproducible, and tied to a command-line task.

## Good Feedback Includes

- what you were trying to accomplish;
- whether the task involved local terminal, remote SSH, file transfer, port forwarding, or Agent co-driving;
- the command or workflow that failed;
- what you expected to happen;
- what actually happened;
- relevant logs, screenshots, or terminal output with secrets removed.

## Bug Reports

For bugs, include:

- operating system and version;
- OpenTerm version or commit;
- steps to reproduce;
- whether the issue happens in a local terminal, remote terminal, or Agent-controlled terminal;
- any relevant error message.

## Feature Requests

For feature requests, describe:

- the workflow you want to improve;
- why existing terminal tools are not enough;
- how the feature should behave when the Agent is active;
- any safety or permission expectations.

## Security Issues

Do not report security issues in public channels. See [SECURITY.md](../SECURITY.md).

# OpenTerm 中文文档

OpenTerm 是一个面向开发者的多终端管理 Agent。它把本地终端、远程主机、文件传输、端口映射和 AI Agent 放在同一个桌面工作台里，让复杂命令行任务更像是和一位会操作终端的搭档协作。

## 解决的问题

开发者电脑里通常有很多 CLI 工具：terminal assistant、terminal assistant、terminal assistant、demo-cli、包管理器、云厂商 CLI、构建工具、部署脚本和各种内部命令。问题不只是“怎么执行一个命令”，而是：

- 多个终端之间频繁切换；
- 本地和远程主机之间上下文断裂；
- 安装、配置、登录、调试过程分散；
- 文件传输和端口映射需要额外工具；
- AI Agent 只能给建议，不能顺滑进入真实终端流程；
- 命令失败后需要人工继续判断下一步。

OpenTerm 让 Agent 进入真实终端工作流。它可以在共驾终端中执行命令、观察输出、继续下一步；你也可以随时暂停、接管或直接手动操作。

## 核心能力

### 多终端管理

OpenTerm 可以同时管理本机和远程终端。终端会话可以按任务组织，也可以切换、恢复、命名、固定，减少窗口和标签页混乱。

### Agent 共驾终端

Agent 不只是聊天回复。它可以在终端中运行命令，根据输出调整下一步，并和你共享同一个操作现场。你可以把它理解为一个可观察、可暂停、可接管的终端协作者。

### CLI 工具编排

OpenTerm 适合驱动各类命令行工具，包括 terminal assistant、terminal assistant、terminal assistant、demo-cli、包管理器、云 CLI、构建脚本和项目命令。Agent 可以帮助你配置、执行、排错和收尾。

### 远程主机工作流

通过 SSH 连接远程主机，并把终端、文件、端口映射和任务上下文组织在同一个工作台里。

### 文件传输和浏览

支持远程文件浏览、上传、下载和文件面板操作，减少在终端、SFTP 工具和文件管理器之间来回切换。

### 端口映射

内置端口转发管理，方便调试远程服务、打开本地预览、连接开发环境。

### 传统终端体验

OpenTerm 不是只能交给 Agent 使用。你依然可以把它当作普通终端来输入命令、查看输出、调整终端尺寸和管理会话。

## 典型使用场景

- 同时启动前端、后端、数据库、日志和构建任务；
- 让 Agent 帮你安装 CLI、配置环境变量、修复依赖问题；
- 连接远程服务器，查看文件并映射端口到本地；
- 让 Agent 观察命令输出并继续排查失败原因；
- 在 Agent 操作时随时接管终端；
- 把多个主机、多个终端和一个任务上下文放在同一个工作台内。

## 工作方式

1. 添加本机或远程主机。
2. 创建终端会话，或让 Agent 为任务创建终端。
3. 向 Agent 描述目标，例如“帮我配置这个 CLI 并跑通项目”。
4. Agent 在共驾终端中执行命令并观察结果。
5. 你可以随时暂停、接管、查看文件、打开端口映射或切换到其他终端。

## 安全与控制

OpenTerm 的目标是增强终端工作流，而不是替你盲目执行所有命令。对于会修改系统状态、访问敏感配置、影响远程服务或删除数据的命令，请先确认上下文和目标。
